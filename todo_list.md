В данном документе подробно описаны шаги, необходимые для полного завершения разработки проекта, устранения разрывов в workflows и интеграции замоканных модулей административной панели с базой данных и бэкенд-сервисами.

## 📅 Шаг 1. Связывание B2B-заказов и Логистики курьеров

В ЛК клиента создаются заказы `DeliveryOrder` с `driverId = null`. Необходимо дать администратору интерфейс для назначения водителей на эти заказы.

### Бэкенд-задачи:
1.  **Добавить эндпоинты управления доставками:**
    *   `GET /api/orders/deliveries` — получить список всех B2B-заказов клиентов (`DeliveryOrder`).
    *   `PATCH /api/orders/deliveries/:id/assign-driver` — назначить водителя (`driverId`) на конкретный заказ.
2.  **Связать с ЛК Курьера:**
    *   Курьер сразу увидит эти заказы у себя на экране «Мои доставки», так как его эндпоинт `/api/courier/orders?driverId=X` отбирает заказы именно по `driverId`.

### Фронтенд-задачи:
1.  **Интеграция [OrdersModule.tsx](file:///c:/Users/User/Desktop/dauren_nan/frontend/src/components/OrdersModule.tsx):**
    *   Удалить mock-данные `MOCK_ORDERS`.
    *   Настроить загрузку заказов через `api.get('/orders/deliveries')`.
    *   Добавить выпадающий список (Select) со списком активных водителей (получать через `api.get('/users?role=DRIVER')`) в строку каждого заказа.
    *   При выборе водителя отправлять `api.patch('/orders/deliveries/:id/assign-driver', { driverId })`.

## 🍞 Шаг 2. Интеграция Планирования Производства

Администратор ставит план, пекарь его выполняет. Нужно связать задания на день с реальной выпечкой.

### Бэкенд-задачи:
1.  **Связать партии пекаря с заданиями производства:**
    *   В `BakerService.startBatch()` проверять, существует ли плановая задача `ProductionTask` для данного продукта на текущую дату.
    *   Если да, то при завершении партии (`BakerService.finishBatch()`) автоматически инкрементировать поле `completed` в таблице `ProductionTask`.

### Фронтенд-задачи:

## 🧪 Шаг 3. Интеграция Технологических карт (Рецептов)
